# Configuration Guide

This guide covers the technical setup for **My Cinema**, including Trakt.tv integration and the configuration of your Amazon Alexa Custom Skill.

## 1. Trakt.tv Integration (Web UI Wizard)

The application features an automated, browser-based authentication flow for Trakt.tv, eliminating the need for manual token management or `curl` requests.

### Authentication Steps:
1.  **Access the Dashboard:** Open your browser and navigate to `http://YOUR_SERVER_IP:5000/setup`.
2.  **API Credentials:** Enter your **Trakt Client ID** and **Client Secret**. (If you do not have these, create an application at the [Trakt.tv API dashboard](https://trakt.tv/oauth/applications)).
3.  **Obtain Authorization PIN:** Click the **"Get PIN"** button provided on the setup page. This will redirect you to Trakt.tv. After authorizing the application, Trakt will provide a numeric PIN code.
4.  **Generate Tokens:** Return to the `/setup` page, paste the PIN into the field, and click **"Generate Tokens"**.
    *   The application will automatically retrieve the `access_token` and `refresh_token`.
    *   These are saved in the persistent `/app/data/trakt_tokens.json` file.
5.  **Persistence:** The `docker-compose.yaml` file maps the local `./data` folder to the container, ensuring your tokens survive container restarts or image updates.

## 2. Amazon Alexa Skill Setup

To bridge your voice commands to the `app.py` backend, you must configure a Custom Skill in the [Alexa Developer Console](https://developer.amazon.com/alexa/console/ask).

### Endpoint Configuration
The application exposes a secure webhook to receive intents from Alexa.
*   **Service Endpoint:** Set this to your public HTTPS URL followed by `/alexa-webhook`.
    *   *Example:* `https://your-domain.com/alexa-webhook`
*   **Security:** Ensure the `ALEXA_SKILL_ID` environment variable in your `docker-compose.yaml` matches the **Application ID** found in your Alexa Skill dashboard. The `app.py` logic will automatically reject any requests that do not originate from this specific skill ID.

### Interaction Model
The project includes predefined speech assets (Intents, Slots, and Utterances) located in the `alexa_speech_assets` folder of the repository.
1.  Navigate to the **JSON Editor** in the Alexa Developer Console.
2.  Import the provided interaction model JSON.
3.  Ensure the following mandatory slots are mapped:
    *   `MovieName`, `MovieYear`
    *   `ShowName`, `Season`, `Episode`
    *   `SourceMode` (Used for "manually" selecting sources)

### Security Requirements
*   **HTTPS:** Amazon requires that the endpoint URL uses a valid SSL certificate. If you are hosting the container behind a reverse proxy (like Nginx, Traefik, or Caddy), ensure it is configured to forward requests to port `5000` on your host machine.
*   **Webhook Validation:** The Python `app.py` handles authentication via:
    ```python
    if incoming_app_id != ALEXA_SKILL_ID:
        return jsonify({"error": "Forbidden"}), 403
    ```
    If you receive "Forbidden" responses, verify that your Skill ID is correctly copied and corresponds to the `ALEXA_SKILL_ID` in your deployment environment.

## 3. Kodi / TMDB Helper Configuration

For voice control to trigger playback correctly, you must verify the pathing of your JSON player files:

*   **Location:** 
    *   **Android:** `/Android/data/org.xbmc.kodi/files/.kodi/userdata/addon_data/plugin.video.themoviedb.helper/players/`
    *   **LibreELEC:** `/storage/.kodi/userdata/addon_data/plugin.video.themoviedb.helper/players/`
*   **Mandatory Files:** Ensure `fenlight_auto.json` and `fenlight_select.json` exist in these directories. If these files are missing, the `PLAYER_DEFAULT` and `PLAYER_SELECT` variables defined in your `docker-compose.yaml` will fail to initiate playback.
