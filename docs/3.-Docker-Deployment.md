# 🚀 Docker Deployment

This application is designed to run in a Docker container. Using `network_mode: host` is highly recommended for Wake-on-LAN functionality (Android TV support) and to ensure smooth communication with the Kodi instance.

## 1. Prerequisites
* **Docker & Docker Compose** installed on your server (e.g., Raspberry Pi, NAS, or Linux server).
* **Kodi** with the **TMDB Helper** add-on installed.
* **Fen Light** add-on installed (the container will handle the necessary patching automatically).

## 2. Prepare the Configuration
Create a project directory and set up the following structure:
```text
/my-cinema
├── docker-compose.yaml
├── data/           # Created automatically
└── adb_keys/       # For Android TV (create empty folder)
```

## 3. Configuration File (`docker-compose.yaml`)

```yaml
version: '3.8'

services:
  kodi-alexa-skill:
    image: ghcr.io/ripleyxlr8/my-cinema-amazon-skill-for-kodi:latest
    container_name: kodi-fenlight-alexa-skill
    restart: unless-stopped
    network_mode: host # Recommended for Wake-on-LAN
    ports:
      - "5000:5000"
    volumes:
      - ./data:/app/data
      - ./adb_keys:/root/.android
    environment:
      - TZ=Europe/Paris
      - TARGET_OS=android # 'android' or 'libreelec'
      - SHIELD_IP=192.168.1.x
      - SHIELD_MAC=AA:BB:CC:DD:EE:FF
      - KODI_PORT=8080
      - KODI_USER=kodi
      - KODI_PASS=kodi
      - TMDB_API_KEY=your_tmdb_api_key
      - ALEXA_SKILL_ID=amzn1.ask.skill.xxxx
      - DEBUG_MODE=false
```

## 4. Environment Variables Reference

Below is the complete list of supported environment variables for the application:

| Variable | Description | Required | Default |
| :--- | :--- | :---: | :--- |
| `TARGET_OS` | The host platform (`android` or `libreelec`). | Yes | `android` |
| `SHIELD_IP` | Local IP address of your Kodi device. | Yes | - |
| `SHIELD_MAC` | MAC address for Wake-on-LAN (Android only). | No | - |
| `SSH_USER` | SSH username for LibreELEC/OpenELEC patching. | No | `root` |
| `SSH_PASS` | SSH password for LibreELEC/OpenELEC patching. | No | `libreelec` |
| `KODI_PORT` | Kodi JSON-RPC Web Server port. | Yes | `8080` |
| `KODI_USER` | Kodi Web Server username. | No | `kodi` |
| `KODI_PASS` | Kodi Web Server password. | No | `kodi` |
| `TMDB_API_KEY` | Your personal TMDB API Key. | Yes | - |
| `ALEXA_SKILL_ID` | Your Alexa Skill ID for security validation. | Yes | - |
| `PLAYER_DEFAULT` | Filename of the default TMDB Helper player. | No | `fenlight_auto.json` |
| `PLAYER_SELECT` | Filename of the manual source TMDB Helper player. | No | `fenlight_select.json` |
| `DEBUG_MODE` | Enable detailed logging (`true`/`false`). | No | `false` |
| `TZ` | Container timezone. | No | `Europe/Paris` |

## 5. Deployment Steps

1.  **Configure Environment:** Fill in your credentials in the `docker-compose.yaml` file provided above.
2.  **Initialize Container:** Run the following command in your project directory:
    ```bash
    docker-compose up -d
    ```
3.  **Verify Status:** Check the logs to ensure the application has started correctly and the API keys are validated:
    ```bash
    docker logs -f kodi-fenlight-alexa-skill
    ```
4.  **Initial Setup:** Navigate to `http://<YOUR_SERVER_IP>:5000/setup` to link your **Trakt.tv** account using the web wizard.
5.  **Health Check:** The container provides a health check endpoint at `http://<YOUR_SERVER_IP>:5000/health`.
