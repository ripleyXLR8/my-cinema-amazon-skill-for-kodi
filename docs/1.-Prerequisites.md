# Prerequisites

Before deploying **My Cinema**, ensure your environment meets the following technical requirements. This project relies on specific connectivity and authentication protocols to interact with your media player and external APIs.

### 1. Hardware & Software Environment
*   **Container Runtime:** [Docker](https://docs.docker.com/get-docker/) with `docker-compose` support.
*   **Network:** The host running the container must be on the same local network as your Kodi device to facilitate API communication (port 8080) and SSH/ADB patching.
*   **Target Device:**
    *   **Android TV (Nvidia Shield):** Requires ADB access enabled in Developer Options.
    *   **LibreELEC / OpenELEC:** Requires SSH access enabled (typically configured in Kodi's service settings).

### 2. External API Credentials
You must obtain the following API keys and tokens:
*   **TMDB API Key:** Obtained via the [TheMovieDB Developer Portal](https://www.themoviedb.org/settings/api).
*   **Trakt.tv API:** You will need a `Client ID` and `Client Secret` from your [Trakt.tv API App settings](https://trakt.tv/oauth/applications). Note: Tokens are generated and managed via the Web UI after deployment.
*   **Alexa Skill ID:** The unique `Application ID` (starting with `amzn1.ask.skill...`) generated when you create your custom skill in the [Alexa Developer Console](https://developer.amazon.com/alexa/console/ask).

### 3. Kodi Configuration
*   **Kodi Web Server:** Must be enabled under **Settings > Services > Control**.
    *   `KODI_PORT`: Defaults to `8080`.
    *   `KODI_USER` / `KODI_PASS`: Required for authenticated JSON-RPC requests.
*   **TMDB Helper Setup:** You must have the [TMDB Helper](https://kodi.wiki/view/Add-on:TMDB_Helper) add-on installed and configured with two specific custom players in the `userdata/addon_data/plugin.video.themoviedb.helper/players/` directory:
    *   `fenlight_auto.json` (for standard playback).
    *   `fenlight_select.json` (with `autoplay=false` to trigger source selection).

### 4. Technical Dependencies (Container Internals)
The Docker image includes the necessary runtime dependencies. If building manually, the project requires:
*   **Python 3.9+**
*   **Flask:** Web interface and API routing.
*   **Requests:** HTTP requests for TMDB, Trakt, and Kodi JSON-RPC.
*   **Wakeonlan:** For sending magic packets to Android devices.
*   **Paramiko:** Required for SSH communication when patching `Fen Light` on LibreELEC/OpenELEC systems.

### 5. Port Requirements
*   **Port 5000 (TCP):** The container must be able to expose this port to receive callbacks from the Alexa Webhook and to serve the Web UI Dashboard.

### 6. File System Permissions
*   **Data Persistence:** The container requires read/write access to a persistent volume (e.g., `./data`) to store `trakt_tokens.json`.
*   **ADB Keys:** If targeting Android, a persistent volume mapping to `/root/.android` is required to store your ADB authorization keys, preventing the need to re-authorize the device after every container restart.
